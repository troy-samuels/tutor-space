# 스프레드시트로 튜터링 비즈니스 관리하는 것을 그만둬야 하는 이유

당신의 Google Sheets가 이렇게 생기지 않았나요?

탭 1: "학생 명단"
탭 2: "학생 명단 (새 버전)"
탭 3: "학생 명단 최종"
탭 4: "학생 명단 최종 (진짜 최종)"

열: 이름, 이메일, 전화번호, 시간대, 레벨, 노트, 결제 상태, 마지막 수업 날짜, 다음 수업 날짜...

그리고 어딘가에 공식이 깨져 있고, 어느 버전이 최신인지 모릅니다.

익숙한 상황인가요? 이 글은 당신을 위한 것입니다.

## 스프레드시트의 문제점

스프레드시트는 무료이고 익숙합니다. 그래서 모든 튜터가 여기서 시작합니다. 하지만 비즈니스가 성장하면서 문제가 시작됩니다.

### 1. 시간 블랙홀

스프레드시트 유지에 드는 시간을 계산해 본 적 있나요?

- 새 학생 추가: 5분
- 수업 후 노트 업데이트: 3분
- 결제 상태 확인: 5분
- 다음 수업 찾기: 2분

학생 20명 × 주 1회 업데이트 = **주 5시간**

월 20시간, 연 240시간—**6주의 근무 시간**을 스프레드시트에 쓰고 있습니다.

### 2. 인간 오류

수동 입력은 실수를 의미합니다:

- 오타로 이메일이 반송됨
- 잘못된 시간대로 수업 혼란
- 결제 누락으로 수입 손실
- 중복 레코드로 혼란

연구에 따르면 스프레드시트의 88%에 오류가 있습니다. 당신의 것도 예외가 아닙니다.

### 3. 통합 부재

스프레드시트는 다른 도구와 자동으로 연결되지 않습니다:

- 학생이 예약하면 → 수동으로 스프레드시트에 추가
- 결제가 들어오면 → 수동으로 상태 업데이트
- 수업이 끝나면 → 수동으로 날짜 기록

모든 것이 **수동**입니다.

### 4. 데이터 분석 불가

스프레드시트에서 이런 질문에 대답하기 어렵습니다:

- 이번 달 수익이 지난 달보다 늘었나요?
- 어느 서비스가 가장 인기 있나요?
- 학생 유지율은 얼마나 되나요?
- 평균 학생 생애 가치는?

차트를 만들 수는 있지만, 그것도 **수동 작업**입니다.

### 5. 확장 불가

학생 5명일 때는 괜찮습니다. 20명이 되면 힘들어지고, 50명이 되면 불가능합니다.

스프레드시트는 확장되지 않습니다—당신이 확장해야 합니다. 그리고 그 시간은 가르치는 데 쓸 수 있는 시간입니다.

## CRM이 필요한 이유

CRM (Customer Relationship Management)은 고객 관계 관리 시스템입니다. 튜터에게는 "학생 관리 시스템"입니다.

### CRM이 제공하는 것

1. **중앙화된 학생 정보**
   - 모든 학생 데이터가 한 곳에
   - 수업 이력 자동 추적
   - 노트와 진행 상황 기록

2. **자동화**
   - 예약이 들어오면 자동으로 학생 레코드 생성
   - 결제가 완료되면 자동으로 상태 업데이트
   - 수업 후 자동으로 기록

3. **통찰력**
   - 대시보드에서 한눈에 비즈니스 현황 파악
   - 수익, 예약, 학생 통계 자동 계산
   - 트렌드와 패턴 식별

4. **시간 절약**
   - 수동 입력 80% 감소
   - 검색 시간 90% 감소
   - 오류 95% 감소

## CRM 옵션

### 범용 CRM (Notion, Airtable)

**장점:**
- 완전 사용자 정의 가능
- 다양한 용도로 사용
- 무료 또는 저렴 ($8-20/월)

**단점:**
- 직접 설정해야 함
- 예약/결제 통합 없음
- 튜터 전용 아님

**적합 대상:**
- 기술에 익숙하고 시간이 있는 튜터
- 다른 비즈니스도 관리하는 경우

### 전문 CRM (HubSpot, Salesforce)

**장점:**
- 매우 강력함
- 많은 통합
- 확장성 높음

**단점:**
- 튜터에게 과함
- 비쌈 ($50-300+/월)
- 학습 곡선 큼

**적합 대상:**
- 대규모 튜터링 회사
- 여러 직원이 있는 경우

### 튜터 전용 플랫폼 (TutorLingua)

**장점:**
- 튜터를 위해 설계됨
- 예약 + 결제 + CRM 통합
- 설정 필요 없음
- 자동 데이터 입력

**단점:**
- 월 비용 ($29-79/월)
- 사용자 정의 제한

**적합 대상:**
- 진지한 독립 튜터
- 올인원 솔루션 원함
- 마켓플레이스에서 전환

## 스프레드시트 vs CRM 비교

| 작업 | 스프레드시트 | CRM (TutorLingua) |
|------|------------|-------------------|
| 새 학생 추가 | 5분 (수동) | 0분 (자동) |
| 수업 노트 기록 | 3분 (탭 전환) | 30초 (인라인) |
| 결제 상태 확인 | 5분 (검색) | 10초 (대시보드) |
| 월간 보고서 | 1시간 (수동 계산) | 0분 (자동) |
| 학생 이력 찾기 | 5분 (스크롤) | 10초 (클릭) |
| **주간 총 시간** | **5시간** | **30분** |

**연간 절약 시간: 230시간 이상**

시간당 ₩50,000이면 = **₩11,500,000 가치**

CRM 비용: ₩350,000-950,000/년

**ROI: 1,200-3,300%**

## 마이그레이션 가이드

스프레드시트에서 CRM으로 전환하는 방법:

### 1단계: 데이터 정리 (1-2시간)

현재 스프레드시트에서:
- 중복 제거
- 오래된 학생 제거/보관
- 필드 이름 통일

### 2단계: CRM 선택 및 설정 (30분-2시간)

- TutorLingua: 10분 설정
- Notion/Airtable: 1-2시간 템플릿 설정
- 전문 CRM: 며칠-주

### 3단계: 데이터 가져오기 (30분-1시간)

대부분의 CRM은 CSV 가져오기를 지원합니다:
1. 스프레드시트를 CSV로 내보내기
2. CRM에서 가져오기
3. 필드 매핑 확인

### 4단계: 새 워크플로우 적용 (즉시)

오늘부터:
- 새 학생은 CRM에만 추가
- 스프레드시트 업데이트 중단
- 모든 노트를 CRM에 기록

### 5단계: 스프레드시트 보관 (1주 후)

1주일 동안 문제 없으면:
- 스프레드시트를 "보관" 폴더로 이동
- 참조용으로만 유지
- 일상 업무에서 사용 중단

## 흔한 반대 의견

### "스프레드시트는 무료인데요"

시간을 돈으로 환산하면 무료가 아닙니다. 월 20시간 × 시급 ₩50,000 = ₩1,000,000/월의 기회 비용.

### "익숙한 방식인데요"

5년 전에 익숙했던 것이 지금도 최선은 아닙니다. 스마트폰도 처음엔 낯설었습니다.

### "설정이 복잡해 보여요"

TutorLingua 같은 튜터 전용 CRM은 10분 만에 설정됩니다. 복잡한 것은 범용 CRM입니다.

### "학생이 몇 명 없어서..."

지금이 전환하기 가장 쉬운 때입니다. 학생이 50명이 되면 마이그레이션이 훨씬 어렵습니다.

## 결론: 미래의 자신을 위한 선물

스프레드시트는 비즈니스 초기에 작동합니다. 하지만 성장하면서 시간을 갉아먹고, 오류를 만들고, 확장을 방해합니다.

CRM으로 전환하면:
- 주 5시간 절약
- 오류 대폭 감소
- 비즈니스 통찰력 확보
- 성장에 집중 가능

가장 좋은 시기는 어제였습니다. 두 번째로 좋은 시기는 **오늘**입니다.
